# Zotero Random Read 使用说明

## 简介

`Zotero Random Read` 用来帮助你在 Zotero 中随机重新打开附件阅读。它不会从整个资料库里盲目抽取，而是只在你选中的 Zotero 文件夹范围里工作，并结合本地阅读记录来提高“未读”或“久未读”文档的出现概率。

## 适用版本

- Zotero `9.x`
- 插件版本 `0.1.0`

## 基本流程

1. 在 Zotero 设置中打开 `随机阅读`。
2. 点击 `添加文件夹`。
3. 在树状选择窗口里勾选要纳入随机池的文件夹。
4. 返回主界面后，点击上方工具栏里的骰子按钮。
5. 插件会随机选择一个条目，并复用 Zotero 的默认打开行为打开附件。

## 文件夹选择规则

- 文件夹选择界面使用树状层级显示父子关系
- 每个文件夹前面都有复选框
- 选中父文件夹时，所有子文件夹会自动一并选中
- 当所有子文件夹都被选中时，父文件夹也会自动视为选中
- 提供 `全选` 和 `全不选`

## 随机策略

插件会参考本机保存的阅读历史，为候选条目分配权重。总体上：

- 没有被随机打开过的条目更容易被抽中
- 很久没有被随机打开过的条目更容易被抽中
- 已经被频繁抽中过的条目权重会下降

这使得随机结果不是完全平均，而是更偏向“重新发现被忽略的文档”。

## 清除阅读数据

设置页中的 `清除阅读数据` 会删除本机保存的随机阅读历史，包括：

- 打开次数
- 最近一次随机打开时间

清除后，插件会把每个文档重新看作“没有随机打开过”的状态。

## 本地数据

插件会把阅读历史保存在 Zotero 数据目录下的插件子目录中，使用本地 JSON 文件存储。该记录默认不跨设备同步。

## 构建产物

本地开发或发布前构建后的 `.xpi` 文件位于：

```text
.scaffold/build/zotero-random-read.xpi
```
